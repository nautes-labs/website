# 维护产品

请确保您已阅读 主体流程 章节，了解部署应用的主体流程和相关术语。

通过维护产品和项目，可以构建与 IT 系统微服务架构一致的管理单元。Nautes 集成了 GitLab，使得维护产品可以在 GitLab 中创建组（group）和默认代码库。产品与 GitLab 中的组（group）保持映射关系，而默认代码库负责存储产品配置清单。

同时，利用 GitLab 的组（group）和项目（project）授权机制进行用户权限管理，有助于限制用户对 GitLab 代码库的访问和操作权限。

支持通过 命令行 和 API 两种方式维护产品。

# 前提条件

# 创建 access token

您需要创建一个 access token，作为请求 API 的请求头。详情参考 注册 GitLab 账号。

# 导入证书

如果您想使用 https 协议访问 Nautes API Server，需要导入证书。

# 创建产品（API）

  1. 访问 Swagger UI，选择 api.product.v1.Product，点击 POST 接口的 try it out，在 product_name 参数中输入产品名称，点击 execute。将生成 API 请求示例，添加 personal access token 作为请求头。
 # 替换变量 $api-server-address 为 Nautes API Server 的访问地址
 # 替换变量 $gitlab-access-token 为 GitLab 访问令牌
 curl -X 'POST' \
 'HTTP://$api-server-address/api/v1/products/product-demo' \
 -H 'accept: application/json' \
 -H 'Content-Type: application/json' \
 -H 'Authorization: Bearer $gitlab-access-token' \
 -d '{
 "git": {
     "gitlab": {	
  	   # 在 GitLab 中创建的产品名称
         "name": "product-demo",  
  	   # 在 GitLab 中创建的产品路径
         "path": "product-demo",
         # 在 GitLab 中创建的产品的可见性，例如：private 或者 public
         "visibility": "private",
         "description": "product-demo"
         }
     }
 }'
  1. 使用 curl 命令或者其他工具（如 Postman、JMeter）执行 API 请求，以新增产品。

请求成功后，将在 GitLab 中生成和产品同名的 group、以及该 group 中名称为 default.project 的代码库。每个产品只有一个 default.project 代码库。default.project 代码库用于存储产品配置清单。

GitLab 中的任何用户都可以创建产品。

# 删除产品（API）

在删除产品之前，请先删除与产品关联的所有实体和资源，例如：项目、代码库、环境和部署运行时等，否则将不能执行删除。

  1. 访问 Swagger UI，选择 api.product.v1.Product，点击 DELETE 接口的 try it out，在 product_name 参数中输入产品名称，点击 execute。将生成 API 请求示例，添加 personal access token 作为请求头。
curl -X 'DELETE' \
'HTTP://$api-server-address/api/v1/products/product-demo' \
-H 'accept: application/json'
-H 'Authorization: Bearer $gitlab-access-token' \
  1. 使用 curl 命令或者其他工具（如 Postman、JMeter）执行 API 请求，以删除产品。

当您的账号是产品成员的 owner 角色或者 GitLab 管理员时，才可以删除产品。

请求成功后，将删除该产品及其相关资源：GitLab 中同名的 group 和 default.project 代码库。

# 查询产品（API）

  1. 访问 Swagger UI，选择 api.product.v1.Product，选择相对路径是 /api/v1/products 的 GET 接口，点击 try it out，点击 execute。将生成 API 请求示例，添加 personal access token 作为请求头。
curl -X 'GET' \
'HTTP://$api-server-address/api/v1/products' \
-H 'accept: application/json'
-H 'Authorization: Bearer $gitlab-access-token' \
  1. 使用 curl 命令或者其他工具（如 Postman、JMeter）执行 API 请求，以查询产品列表。产品列表的返回值示例如下：
{
  "items": [
      {
          "name": "api-server-test",
          "git": {
              "gitlab": {
                  "path": "api-server-test",
                  "visibility": "private",
                  "description": ""
              },
              "github": null
          }
      },
      {
          "name": "demo-0410",
          "git": {
              "gitlab": {
                  "path": "demo-0410",
                  "visibility": "private",
                  "description": "demo-0410"
              },
              "github": null
          }
      },
      ......
  ]
}

当您的账号是产品成员，可以查询到被授权的相关产品。

# 查看产品详情（API）

  1. 访问 Swagger UI，选择 api.product.v1.Product，选择相对路径是 /api/v1/products/{productName} 的 GET 接口，点击 try it out，在 product_name 参数中输入产品名称，点击 execute。将生成 API 请求示例，添加 personal access token 作为请求头。
curl -X 'GET' \
'HTTP://$api-server-address/api/v1/products/product-demo' \
-H 'accept: application/json'
-H 'Authorization: Bearer $gitlab-access-token' \
  1. 使用 curl 命令或者其他工具（如 Postman、JMeter）执行 API 请求，以查询产品详情。产品详情的返回值示例如下：
{
  "name": "api-server-test",
  "git": {
      "gitlab": {
          "path": "api-server-test",
          "visibility": "private",
          "description": ""
      },
      "github": null
  }
}

当您的账号是产品成员，可以查询到被授权的产品详情。
