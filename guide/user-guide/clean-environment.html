# 清理环境

本文档将描述一个用户应用的部署运行环境的清理过程。包括以下步骤：

销毁运行环境
销毁集群
销毁环境

开始本文档的操作之前，请确保运行集群中成功部署了应用的部署运行环境。

# 销毁运行环境

下文将描述使用命令行销毁产品运行环境相关的 Kubernetes 资源文件。

  1. 将命令行程序的代码库克隆到本地。
git clone https://github.com/nautes-labs/cli.git
  1. 替换位于相对路径 examples/demo-product.yaml 下配置运行环境模板的变量，包括 $suffix，$runtime-cluster。

运行环境模板及其注释，详情参考 准备运行环境。

  1. 下载 命令行工具 (opens new window)，执行以下命令，将删除 GitLab 代码库中的 Kubernetes 资源文件。您可以执行 nautes -h、nautes apply -h 等命令来查看帮助说明。
# 删除用于部署产品的运行环境
nautes remove -f examples/demo-product.yaml -t $gitlab-access-token -s $api-server-address

GitLab 账号的注意事项，与 准备运行环境 的注意事项相同。
命令执行成功后，将根据 Kubernetes 资源文件删除GitLab 中和产品有映射关系的 group 及其代码库，并在运行集群上销毁部署产品的运行环境。

# 销毁集群

您需要根据在 部署一个应用 中注册的集群形态，决定销毁哪种形态的集群。

# 销毁物理集群

请确保已成功注册物理集群。

  1. 将命令行程序的代码库克隆到本地。
git clone https://github.com/nautes-labs/cli.git
  1. 替换位于相对路径 examples/demo-cluster-physical-worker.yaml 下注册物理集群的模板的变量，包括 $suffix、$api-server 和 $kubeconfig。

注册物理集群的模板及其注释，详情参考 注册物理集群。

  1. 下载 命令行工具 (opens new window)，执行以下命令，以销毁物理集群。
# 销毁物理集群
nautes remove -f examples/demo-cluster-physical-worker.yaml -t $gitlab-access-token -s $api-server-address

GitLab 账号、API Server 的注意事项，与 注册物理集群 的注意事项相同。
命令执行成功后，将会销毁物理集群。

# 销毁虚拟集群

请确保已成功注册虚拟集群。

  1. 将命令行程序的代码库克隆到本地。
git clone https://github.com/nautes-labs/cli.git
  1. 替换位于相对路径 examples/demo-cluster-virtual-worker.yaml 下的注册虚拟集群的模板的变量，包括 $suffix、$api-server、$host-cluster 和 $api-server-port。

注册虚拟集群的模板及其注释，详情参考 注册虚拟集群。

  1. 下载 命令行工具 (opens new window)，执行以下命令，将销毁虚拟集群。
# 销毁虚拟集群
nautes remove -f examples/demo-cluster-virtual-worker.yaml -t $gitlab-access-token -s $api-server-address
  1. 替换位于相对路径 examples/demo-cluster-host.yaml 下的注册宿主集群的模板的变量，包括 $suffix、$api-server 和 $kubeconfig。

注册宿主集群的模板及其注释，详情参考 注册宿主集群。

  1. 执行以下命令，以销毁虚拟集群所属的宿主集群。
# 销毁宿主集群
nautes remove -f examples/demo-cluster-host.yaml -t $gitlab-access-token -s $api-server-address

命令执行成功后，将会销毁虚拟集群及所属的宿主集群。
