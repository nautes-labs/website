# 注册运行时集群

在开始本节之前，请确保您已阅读 主体流程 章节，了解部署应用的主体流程和相关术语。

运行时集群用于承载应用的运行时环境。集群形态支持物理集群和虚拟集群。

支持通过 命令行 和 API 两种方式注册运行时集群。

# 前提条件

# 创建 access token

您需要创建一个 access token，作为请求 API 的请求头。详情参考注册 GitLab 账号。

# 导入证书

如果您想使用 https 协议访问 Nautes API Server，请导入证书。

# 注册物理集群（API）

  1. 通过接口定义 Cluster_SaveCluster 生成 API 请求示例，并添加 access token 作为请求头。
	# 替换变量 $api-server-address 为 Nautes API Server 的访问地址
	# 替换变量 $gitlab-access-token 为 GitLab 访问令牌
	# $cluster_name，集群名称    
    curl -X 'POST' \
      'HTTP://$api-server-address/api/v1/clusters/$cluster_name' \
      -H 'accept: application/json' \
      -H 'Content-Type: application/json' \
      -H 'Authorization: Bearer $gitlab-access-token' \
      -d '{
      # 集群的 API SERVER URL
      "api_server": $api_server,
      # 集群种类：目前只支持 kubernetes
      "cluster_kind": "kubernetes",
      # 集群类型：virtual或physical
      "cluster_type": $cluster_type,
      # 集群用途：host或worker
      "usage": $usage,
      # argocd 域名：$cluster_name 替换为集群名称,$cluster_ip 替换为集群IP
      "argocd_host": "argocd.$cluster_name.$cluster_ip.nip.io",
      # traefik 配置
      "traefik": {
        "http_node_port": "30080",
        "https_node_port": "30443"
      },
      # 集群的 kubeconfig 文件内容：$kubeconfig 替换为物理集群的 kubeconfig
      "kubeconfig": $kubeconfig
    }'

# 删除物理集群（API）

请确保物理集群注册成功。

在删除集群之前请先删除产品配置清单。详情参考 删除产品配置清单（命令行） ，或者 维护部署运行时 、维护环境、 维护代码库、维护项目、维护产品 中的删除章节（API）。

  1. 通过接口定义 Cluster_DeleteCluster 生成 API 请求示例，并添加 access token 作为请求头。
    curl -X 'DELETE' \
      'HTTP://$api-server-address/api/v1/clusters/$cluster_name' \
      -H 'accept: application/json' \
      -H 'Authorization: Bearer $gitlab-access-token'

替换变量后的请求示例如下：

    curl -X 'DELETE' \
      'HTTP://xxx.xxx.xxx.xxx:xxxxx/api/v1/clusters/cluster-physical' \
      -H 'accept: application/json' \
      -H 'Authorization: Bearer xxxxxxxxxxxxxxxxxxxx'
  1. 使用 curl 命令或者其他工具执行 API 请求。
    请求成功后，将删除物理集群，以及在租户配置库的集群资源文件。

只有当您的账号是租户配置库的成员，并且有 main 分支的写入权限，才可以删除运行时集群。

# 删除虚拟集群（API）

请确保虚拟集群注册成功。

在删除集群之前请先删除产品配置清单。

  1. 通过接口定义 Cluster_DeleteCluster 生成 API 请求示例，并添加 access token 作为请求头。API请求的格式与删除物理集群相同。 请求示例如下：
    curl -X 'DELETE' \
      'HTTP://xxx.xxx.xxx.xxx:xxxxx/api/v1/clusters/cluster-virtual' \
      -H 'accept: application/json' \
      -H 'Authorization: Bearer xxxxxxxxxxxxxxxxxxxx'

  1. 使用 curl 命令或者其他工具执行 API 请求。请求成功后，将删除虚拟集群。

  2. 通过接口定义 Cluster_DeleteCluster 生成 API 请求示例，并添加 access token 作为请求头。API请求的格式与删除物理集群相同。 请求示例如下：

    curl -X 'DELETE' \
      'HTTP://xxx.xxx.xxx.xxx:xxxxx/api/v1/clusters/cluster-host' \
      -H 'accept: application/json' \
      -H 'Authorization: Bearer xxxxxxxxxxxxxxxxxxxx'
  1. 使用 curl 命令或者其他工具执行 API 请求。请求成功后，将删除宿主集群。

只有当您的账号是租户配置库的成员，并且有 main 分支的写入权限，才可以删除运行时集群。
